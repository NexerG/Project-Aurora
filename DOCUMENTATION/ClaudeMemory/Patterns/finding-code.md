# Pattern — finding code via NAMESPACES.md

`NAMESPACES.md` (repo root) is an **auto-generated** index of every namespace → relative file
path, sorted unique. Generated by `_Build/GenerateNamespaces.cmd`, which scans all `*.cs`
files (excluding `\.claude\`) for `^namespace ` lines.

## Use it to locate code

1. Know the namespace (e.g. `ArctisAurora.EngineWork.Rendering`) or guess it from the system.
2. Look it up in `NAMESPACES.md` to get the file path(s) — many namespaces span several files.
3. Open the specific file rather than blind-searching the tree.

## Keep it fresh

- It is generated output: **do not hand-edit**.
- If files were added/moved/renamed and the map looks stale, regenerate:
  `_Build/GenerateNamespaces.cmd` (run from `_Build`, writes `NAMESPACES.md` at repo root).
- One namespace commonly maps to many files; one folder may contain multiple namespaces.

## Why memory should not hardcode paths

Paths in this codebase move. Recording `namespace + ClassName` and resolving through
`NAMESPACES.md` survives refactors; recording absolute paths rots. This is why ClaudeMemory
files reference symbols, not paths.
